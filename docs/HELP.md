简介
一款能够在飞书上使用终端和agent-cli的工具

核心竞争力
1. 低成本：不用api key方式的高token消耗AI用法，直接用你本地安装的agent，比如免费的opencode
2. 控制终端：控制了终端就等同于控制了整个PC或云服务器
3. 控制本地agent：一键选择本地agent，启动会话直接启用agent, 任何不限于claude code/codex/gemini，可以是你们公司自研的agent-cli产品，保障数据安全
4. 多会话管理：一个会话一个群聊，可在不同的群聊中做不同的任务，而且可以多人在同一个群聊中操控同一个agent干活
   想象一下，你的需求在测试阶段遇到bug，你让测试直接将问题抛到群聊里，你的本地的agent收到群聊消息后自动解决问题，画面不要太美好
5. 实时查看进展：在一轮任务前，你可以手动或自动刷新飞书推送过来的进展，任务完成后会再次发送消息通知你，你可以让他艾特你，因为默认不艾特的消息不会发通知
6. web终端&飞书共通：web端嵌入真实终端，配置文本输入框解决终端输入困难问题，支持添加快捷键，飞书和终端双向控制且共享会话
7. 轻量级飞书配置：一键扫码创建飞书机器人
8. 飞书快捷键：担心飞书上做终端命令操作困难，不存在的，在飞书上发送消息后立马会回复一个消息卡片，上面配置有快捷键，有系统快捷键 esc/enter/ctrl-c等，你还可以自定义快捷命令
9. 指令预设-高阶用法
1. 会话名预设：如果开启了一个名为 xx 的会话，你可以给他预设命令，启动的前直接执行命令，比如你希望能够在不同的目录下工作，那就可以预设进到某一个目录，也可以在进到目录后启动你的agent，可以是任何的agent，claude code、codex、gemini等等
2. 会话启动命令：你可以指定任何一个会话启动前要执行的命令，比如你想使用zsh，那直接设置使用zsh的命令
   安装&使用
   开源地址：https://gitee.com/eleven_lj/easy_terminal.git
   一键安装：npm install -g @lijuneleven/easy-terminal
   启动：
1. easy-terminal
   默认8080端口，网页访问：localhost:8080
2. easy-terminal  -p 8888
   指定端口号
3. easy-terminal  -p 8888 --config-dir xxxx
   指定配置文件目录，默认～/.easy-terminal/conf
   必要配置：
1. 选择本地agent，可以不选，自己单独添加agent启动快捷键或者会话预设agent启动命令，建议选上
2. 飞书配置：飞书扫码一键配置
