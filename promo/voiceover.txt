比 OpenClaw、Hermes 更接地气的 Agent，应该是什么样？

我的答案不是再做一个新的智能体，而是先控制终端。

因为真正工作的 Agent，本来就跑在终端里。

OpenCode、Codex CLI、Claude Code、Gemini CLI，甚至你自己的脚本和本地模型，只要能在终端中运行，就可以被 Easy Terminal 远程使用。

这和云端 Agent 最大的区别是：它不是把任务丢到一个黑盒里等结果，而是让 Agent 直接跑在你的本机或服务器环境里。

成本更低，因为你可以使用免费的、开源的、本地的 CLI Agent。

能力更强，因为终端能做什么，Agent 就能做什么。启动服务、看日志、跑测试、改代码、执行脚本，都在同一个工作流里。

体验也更接地气。你可以在飞书里实时看到输出，中途补充指令，随时接管；也可以在 Web 端管理多个终端会话，让服务、日志、测试和 Agent 并行运行。

所以 Easy Terminal 不是又一个 Agent。

它是所有终端 Agent 的入口。

控制终端，就等于控制所有能在终端里运行的智能体。
